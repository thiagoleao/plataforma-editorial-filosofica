# ADR-021 - Refresh de Design

**Status:** Implementada (2026-07-16)

## Contexto

A `editorial-ui` (ADR-016) foi construída com Tailwind puro e componentes HTML simples — funcional, mas visualmente básica. O usuário pediu explicitamente uma UI "moderna e bonita" para a plataforma, principalmente porque as ADR-018 a 020 vão adicionar telas novas (login, sugestões, editor de manuscrito) que se beneficiam de um sistema de design mais consistente desde o início, em vez de nascerem no mesmo estilo básico e precisarem de retrabalho depois.

## Decisão original (Proposta, 2026-07-15) — substituída

A versão original desta ADR propunha adotar shadcn/ui (Radix + Tailwind, componentes copiados para o repositório) e um rollout distribuído dentro das ADR-018/019/020, sem fase própria. Isso foi **substituído** pela decisão abaixo antes de qualquer implementação — nenhum código shadcn/ui chegou a ser escrito.

## Decisão final (2026-07-16)

Ao implementar, o usuário deu uma direção mais específica e concreta do que o desenho original previa: **reaproveitar o sistema visual já validado no "Escritório de Arquitetura"** (projeto irmão no mesmo workspace, `AI-PLATFORM/ESCRITÓRIO DE ARQUITETURA`), em vez de introduzir uma biblioteca de componentes nova. Justificativa do próprio usuário: não reinventar a roda — o outro projeto já tem um visual testado e aprovado.

1. **Sistema "glass" (glassmorphism) portado quase literalmente** do `globals.css` do Escritório de Arquitetura: cards com `backdrop-blur`, fundo com foto + gradiente por trás, paleta índigo, pílulas arredondadas para botões, inputs com blur. Implementado como classes utilitárias Tailwind (`@layer components` — `glass-card`, `glass-pill`, `glass-input`, `glass-alert-*`, `glass-bar`, `glass-item`, `glass-link`), não uma lib de componentes copiados — mais simples que o shadcn/ui original e já provado em produção no projeto irmão.
2. **Identidade visual própria com os assets que o usuário forneceu** (pasta `imagens/` na raiz do repo): logo completo, ícone (medalhão dourado com o símbolo Φ) e uma foto de biblioteca clássica como fundo — substituindo o `fundo-escritorio.jpg` genérico do projeto irmão por uma imagem que já é do próprio livro/plataforma.
3. **Renomeação de marca: "Curadoria Editorial" sai, "Plataforma Editorial Filosófica" entra em 100% dos lugares** — título da aba, cabeçalho, metadata. Não é mais um nome secundário exibido ao lado do principal (como era antes); é o nome único da aplicação.
4. **Navegação lateral persistente** (Projetos / Sugestões) substituindo o link único no cabeçalho — mantém o espírito do item 5 do desenho original, mas usando o padrão de sidebar do Escritório de Arquitetura (`lucide-react` para ícones) em vez de um componente shadcn/ui.
5. **Dark mode**: mantido via `prefers-color-scheme` nativo (já existia na `editorial-ui` antes desta ADR) — todas as classes `glass-*` têm variante `dark:`. Não foi adotado `next-themes`/toggle manual (item 3 do desenho original) porque o padrão do projeto irmão também não usa — consistência entre os dois projetos do workspace pesou mais que a funcionalidade extra.
6. **Toasts (item 4 do desenho original) não foram implementados.** O padrão de feedback já em uso (`<p>` de mensagem após uma Server Action) foi mantido, só restilizado com `glass-alert-info`/`glass-alert-error` — trocar por toasts exigiria uma biblioteca nova sem ganho claro dado o restante do design já ficar consistente.

## Consequências

- Reduz o risco de inconsistência visual entre as telas antigas e as novas, e também entre os dois projetos do workspace (`editorial-ui` e o Escritório de Arquitetura passam a compartilhar a mesma linguagem visual).
- Rollout foi feito de uma vez em todas as telas existentes (não distribuído entre ADRs futuras, diferente do plano original) — na prática, como o design refresh acabou sendo pedido depois que ADR-019/020 já estavam prontas, não havia mais "próximas ADRs" para carregar o rollout; fazer tudo de uma vez foi mais simples e evitou um período prolongado de inconsistência visual.
- Custo zero de dependência nova de peso (só `lucide-react`, ícones); nenhuma lib de componentes adicionada.
- O acervo de imagens da marca (`imagens/` na raiz do repo) fica fora do `editorial-ui` — os arquivos otimizados para uso web vivem em `editorial-ui/public/images/`, gerados a partir dos originais.

## Nota de implementação (2026-07-16)

**Implementada em todas as telas existentes da `editorial-ui`.**

- Assets: `imagens/Plataforma Editorial Filosófica - {Logo grande,Icon,fundo}.png` (fornecidos pelo usuário, 14MB–971KB, resoluções até 3840×2160) redimensionados/comprimidos via `sips` para `editorial-ui/public/images/` — `logo.png` (960px, wordmark completo, reservado para contextos espaçosos), `icon.png` (512px, medalhão Φ, usado na sidebar e como `app/icon.png`/favicon), `fundo-biblioteca.jpg` (2000px, JPEG qualidade 55, ~530KB — o PNG original de 14MB nunca teria sentido como `background-image` de página).
- `app/globals.css`: reescrito com o sistema `glass-*` portado do Escritório de Arquitetura (mesma estrutura, paleta índigo idêntica — já combinava com a cor da logo nova). `--bg-photo` aponta para `fundo-biblioteca.jpg`.
- `components/Sidebar.tsx` (novo): ícone Φ (56px) + "Plataforma Editorial Filosófica" por extenso, nav com `lucide-react` (`BookOpen` para Projetos, `Sparkles` para Sugestões). **Decisão de design**: o wordmark completo (`logo.png`) não coube bem numa sidebar estreita com texto legível (é um card 16:9 com texto em 3 linhas) — optou-se pelo ícone compacto + texto HTML real em vez de forçar a imagem inteira num espaço apertado.
- `app/layout.tsx`: sidebar + main, título/metadata trocados para "Plataforma Editorial Filosófica". Nenhuma outra ocorrência de "Curadoria Editorial" restava no código (verificado via busca no repo inteiro).
- Todas as páginas (`/`, `/projects/[id]`, `/projects/[id]/duplicate-report`, `/projects/[id]/chapters/[id]`, `/projects/[id]/chapters/[id]/manuscript`, `/chapter-suggestions`, `/chapter-suggestions/[id]`) e componentes (`ChapterBuilder`, `ConceptPicker`, `SourceResultCard`, `StatusBadge`, `SegmentInsightPanel`, `TransitionEditor`, `ManuscriptEditor`, `LiteralSegmentNode`) migrados de `rounded-md border border-gray-*` / `bg-black` para as classes `glass-*` correspondentes — nenhuma mudança de lógica, só `className`.
- Testado localmente (`npm run dev`) e via deploy real em Cloud Run (revisão `editorial-ui-00014-zjh`) — light e dark mode, todas as telas principais (lista de projetos, capítulo, manuscrito, sugestões), sem erros de console.
